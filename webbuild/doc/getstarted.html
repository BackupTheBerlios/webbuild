Getting started with WebBuild

System Requirements

Please not that you need a Cygwin environment on Microsoft Windows Systems, otherwise WebBuild does not work. On UNIX-based operating systems it should run native if you have installed the following tools:

  • Streaming Editor sed
  • GNU AWK
  • GNU make

Hint

If you want to use WebBuild properly you have to keep to some rules concerning the internal file structure of your Web project. In this example we assume that you have a project called "foobar" that is located under /home/itsme/web/foobar.

Extracting Package

You have downloaded the ZIP file from our server. Now put it to your projects directory, e.g. /home/itsme/web/foobar and extract it with your common decompression program.

Now there is a new folder created, named webbuild. This folder has two sub-folders system and doc. The folder doc contains some HTML files with further information and helpful instructions how to handle WebBuild.

System is the more important folder. This is the one which contains the main files which will build your pages later on. You have to copy Makefile and Makefile.rules to the root directory of your project. Currently the structure of your project should look like this:

/home/itsme/web/foobar
  |-webbuild
  +--system
Makefile
Makefile.rules

Creating Source Directory

The source directory (src) is the one which inherits the files with the content you want to build HTML pages from. These files contain your defined patterns and the text or images you want to place into your template file.

You should create the source directory named src directly under your root directory.

Creating Template File

Using WebBuild requires a template file. This file is written in HTML code and contains the pattern that you want to replace with the content from the source directory. You have to place the template under the root directory.

These tags look like <hive field="tag-name" />. Replace "tag-name" with a qualified name containing letters and numbers.

Now your projects directory should look like this:

/home/itsme/web/foobar
  |-src
  |-webbuild
  +--system
Makefile
Makefile.rules
template.html

Creating Source File

As mentioned before you have to create a source file that contains the content you want to replace in the HTML template. Your name the source file like the HTML file you want to create but instead you use the ending *.src, e.g. index.html will be index.src.

The content should look like this <hive field="tag-name">My content, I want to replace</hive> . Save your file under the former created directory src.

Configuring the Make-System

Before you can build a HTML page you have to open and edit Makefile.rules. Makefile.rules contains two options:

  • ROOT
  • ADDON

You have to set ROOT with the global file path to your project, e.g. /home/itsme/web/foobar.

You have to set ADDON with the name of the WebBuild directory with the additional files. If you did not rename the directory you can leave the name unchanged.

Executing Make-System

See commands.html with a list of commands you can use. There are also additional information and tips.
